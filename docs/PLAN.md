# StayWithMeForever · 项目规划总纲

## 一、项目定位

**轻量级第一人称数字人格构建系统**（Lightweight First-Person Persona Distillation System）。

区别于 nuwa-skill 从公开网络资料反推名人认知模式，本项目通过**结构化问卷**直接采集身边人的一手认知数据。

### 核心差异化
- 不依赖服务器，数据完全本地存储（IndexedDB）
- 不依赖公开网络资料，用一手问卷直接建模
- 支持人格版本回溯（记录"2024年的我" vs "2026年的我"）
- 输出标准化 Agent Skills 协议 SKILL.md

## 二、产品形态

- **A 模式**：纯工具 - 答题后下载 SKILL.md，导入外部 Agent Runtime
- **B 模式**：一站式应用 - 内置聊天界面，直接在网页对话

本产品同时支持这两种模式。

## 三、目标用户与场景

### 用户
- 想创建身边人 AI 分身的普通用户
- 想记录自己不同时期状态的人
- 想通过问卷更好了解朋友的人

### 优先场景
1. **自我记录**：不同时间段的自己可回访、可对话
2. **朋友蒸馏**：为朋友创建 AI 人格
3. **后期可拓展**：伴侣分析、团队协作图谱

## 四、人格建模五层结构

| 层级 | 名称 | 提取方式 | 输出 |
|------|------|---------|------|
| L1 | 基础人格 Trait | 自填 MBTI + 基础自评 | 行为倾向标签 |
| L2 | 认知模式 Cognitive | 情境选择题（25题） | 决策风格、信息处理方式 |
| L3 | 价值观 Value | 两难排序题（20题） | 价值优先级、取舍倾向 |
| L4 | 表达DNA Expression | 开放简答题（6题）+ 风格选择题（25题） | 句式、幽默、情感表达 |
| L5 | 关系动态 Relationship | 社交场景题（20题） | 不同角色的行为模式 |

## 五、技术架构

### 全栈纯前端
```
┌──────────────────────────────────────┐
│        浏览器（React PWA）            │
│  ┌──────┐ ┌─────────┐ ┌───────────┐ │
│  │ 问卷  │ │ SKILL.md│ │ 内置聊天   │ │
│  │ 引擎  │ │ 生成器   │ │ (SystemP) │ │
│  └──────┘ └─────────┘ └───────────┘ │
│  ┌──────────────────────────────────┐ │
│  │   IndexedDB (Dexie.js)          │ │
│  │   Persona / Version / Chat       │ │
│  └──────────────────────────────────┘ │
│  ┌──────────────────────────────────┐ │
│  │   LLM Adapter                    │ │
│  │   DeepSeek / OpenAI / Ollama     │ │
│  └──────────────────────────────────┘ │
└──────────────────────────────────────┘
```

### LLM 调用
所有 LLM 调用由浏览器**直接**发往提供商（或本地 Ollama），不经过任何中间服务器。

### 优先级
1. DeepSeek (`deepseek-chat`)
2. OpenAI (`gpt-4o`)
3. Anthropic (`claude-sonnet-4-20250514`)
4. Ollama (本地，`llama3`)

## 六、SKILL.md 生成管道

两阶段生成：
1. **阶段一 - Analyzer**：问卷 JSON → 结构化人格档案
   - 心智模型（3-5个）
   - 决策启发式（5-8条）
   - 表达 DNA 规则
   - 价值观与反模式
   - 诚实边界
2. **阶段二 - Generator**：人格档案 JSON + nuwa-skill 模板 → SKILL.md

## 八、关键设计决策

1. **纯前端**：无服务器、零成本、隐私绝对安全
2. **版本管理是核心**："记录不同时间段的自己"是产品核心价值
3. **复用 nuwa-skill 协议**：输出标准 SKILL.md，兼容生态
4. **Prompt 本地化**：Analyzer 和 Generator Prompt 用英文（LLM 指令更精确），Skill 内容用中文
5. **DeepSeek 优先**：中文理解能力好，价格低，用户明确指定

## 九、GLOSSARY

| 术语 | 定义 |
|------|------|
| Persona | 人格主体，对应一个人 |
| PersonaVersion | 人格版本，一个 Persona 可以有多个时间点的版本 |
| SKILL.md | Agent Skills 协议的标准 Skill 文件，Markdown + YAML frontmatter |
| Analyzer | LLM 分析阶段，从问卷数据中提取结构化人格特征 |
| Generator | LLM 生成阶段，将人格特征填充到 SKILL.md 模板 |
| Expression DNA | 语言表达的风格特征（句式、幽默、情感等） |
| 诚实边界 | Skill 的局限性声明 |
